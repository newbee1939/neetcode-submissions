## ゴール

問題を見た瞬間に解法パターンが浮かび、スラスラ解ける状態。
個別の問題を暗記するのではなく、**「問題の特徴 → 解法パターン」の逆引き**を鍛える。

## 進め方

[NeetCode 150](https://neetcode.io/practice) をロードマップ順に解く

### 1問のサイクル

1. 10分は自力で考える（タイマー必須）
2. 解けなければ解説を**理解できるまで**見る
3. 見ずに白紙から再実装する ← ここが本番
4. 詰まった問題だけ、ファイル冒頭に「なぜこの解法を選べるか（Trigger）」を1行コメントで残す

コードには解法（答え）しか残らない。`Trigger`（問題文 → 解法の逆引き）だけは
コードから読み取れないので、つまずいた問題に限って記録する。

## 続けるコツ

- **解けなくて当たり前**。詰まった問題＝弱点パターンの発見であり、伸びしろ
- 早く答えを見すぎない。読むだけで満足しない（必ず再実装）
- 解いた問題は **1日後・1週間後・1ヶ月後** に解き直す
- 制約から計算量を見積もる癖をつける
